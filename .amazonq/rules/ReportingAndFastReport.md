# Reporting and FastReport Rules – Accounting WebApp Project
**Version:** 1.0 | **Effective Date:** March 2026  
**Reporting Engine:** FastReport.OpenSource 2026.1.4 + FastReport.OpenSource.Web 2026.1.4 + FastReport.OpenSource.Export.PdfSimple 2026.1.4  
**Primary Goal:** Generate accurate, VAS-compliant financial statements (Balance Sheet, Income Statement, Cash Flow Statement, Trial Balance, General Ledger extracts, Notes) in a server-rendered MVC environment. Ensure reports match Circular 99/2025/TT-BTC appendices, support VND formatting, comparative periods, and audit traceability.

## 1. Core Principles for Reporting
- All reports MUST be **VAS-statutory compliant** first (reference VASCompliance.md): exact structure/titles from Circular 99 appendices, grouped by account type/level, assets = liabilities + equity, indirect cash flow method default.
- Use **server-side rendering** via WebReport in Razor views → no client-side report logic.
- Prioritize **PDF export** (via PdfSimple plugin) for official/statutory filings.
- Support **comparative figures** (current vs. prior period/year) where required (e.g., Balance Sheet).
- Currency: Default VND with Vietnamese formatting (1.234.567 ₫) – use resource files or custom formatters.
- Data sources: Aggregated DTOs/lists from EF Core queries (projections for performance) → register via `Report.RegisterData(...)`.
- Auditability: Log report generation (user, params, timestamp) via Serilog; include "Generated by [User] on [Date]" footer in reports.

## 2. NuGet Packages (Pin Versions)
Always use exact versions for stability:
- FastReport.OpenSource → 2026.1.4
- FastReport.OpenSource.Web → 2026.1.4
- FastReport.OpenSource.Export.PdfSimple → 2026.1.4 (for free PDF export)

Add to Web.csproj or via Directory.Build.props.

## 3. Report File Management (.frx)
- Store templates in: `wwwroot/Reports/` (e.g., `BalanceSheet.frx`, `IncomeStatement.frx`)
- Naming: `{ReportName}.frx` (PascalCase, English for code consistency; Vietnamese labels inside designer)
- Design tool: Use free FastReport Designer (download from GitHub releases: v2026.1.0 or later)
- VAS rules in designer:
  - Use bands/tables matching Circular 99 layouts (e.g., assets current/non-current sections)
  - Add expressions for totals/sums (e.g., [Sum(Amount) where Type=Asset])
  - Include required Notes section (text band with placeholders)
  - Embed fonts if needed (VND symbols); test PDF output
- Version control: Commit .frx files to Git; avoid binary diffs issues by using text-diff friendly tools if possible

## 4. Integration Pattern in MVC (Traditional)
- **Service layer**: ReportService.cs handles data aggregation via EF Core
  - Example method:
    ```csharp
    public async Task<List<BalanceSheetRowDto>> GetBalanceSheetAsync(DateTime from, DateTime to)
    {
        return await _context.Accounts
            .AsNoTracking()
            .Where(a => a.Level <= 2)
            .Select(a => new BalanceSheetRowDto 
            { 
                Code = a.Code, 
                Name = a.Name, 
                Balance = a.TransactionEntries
                    .Where(te => te.Journal.Date >= from && te.Journal.Date <= to)
                    .Sum(te => te.Debit.Amount - te.Credit.Amount)
            })
            .ToListAsync();
    }
    ```
- **Controller**: ReportsController.cs – thin, calls service and passes data to view
  - Example action:
    ```csharp
    [HttpGet]
    public async Task<IActionResult> BalanceSheet(DateTime? from, DateTime? to)
    {
        var data = await _reportService.GetBalanceSheetAsync(
            from ?? DateTime.Today.AddYears(-1), 
            to ?? DateTime.Today
        );
        return View(data);  // Pass DTO list to view
    }
    ```
- **View** (e.g., Views/Reports/BalanceSheet.cshtml):
  ```html
  @model List<BalanceSheetRowDto>

  <div class="card">
      <div class="card-header">Báo cáo Tình hình Tài chính (VAS 01)</div>
      <div class="card-body">
          @Html.Partial("_FastReportPreview", new WebReport { /* ... */ })
      </div>
      <div class="card-footer">
          <a href="@Url.Action("ExportPdf", new { /* params */ })" class="btn btn-primary">Xuất PDF</a>
      </div>
  </div>
  ```
- **FastReport WebReport**: ReportService prepares data (DTOs), controller creates WebReport and sets RegisterData, view renders preview
- **Export action** (PDF):
  ```csharp
  public async Task<IActionResult> ExportPdf(DateTime? from, DateTime? to)
  {
      var data = await _reportService.GetBalanceSheetAsync(...);
      var webReport = new WebReport();
      string reportPath = Path.Combine(_env.WebRootPath, "Reports/BalanceSheet.frx");
      webReport.Report.Load(reportPath);
      webReport.Report.RegisterData(data, "BalanceData");

      using var pdfExport = new PdfSimpleExport();
      using var stream = new MemoryStream();
      webReport.Report.Export(pdfExport, stream);
      stream.Position = 0;

      return File(stream.ToArray(), "application/pdf", $"BalanceSheet_{DateTime.Today:yyyyMMdd}.pdf");
  }
  ```

## 5. Limitations of OpenSource Edition (Know & Mitigate)
- PDF export (PdfSimple): Basic – no encryption, digital signatures, advanced font embedding, or forms. Mitigate: Use for draft/statutory previews; upgrade to paid FastReport .NET if signing needed later.
- No Excel/HTML advanced exports in free version (only image/HTML basic).
- No built-in online designer (use desktop FastReport Designer).
- Performance: Large reports → use EF projections, avoid loading full datasets; paginate previews if needed.
- Browser support: Latest Chrome/Edge/Firefox (test on IE/Edge Legacy if legacy users).

## 6. VAS-Specific Report Rules
- Balance Sheet: Group by 1xx (Assets), 2xx (Liabilities? Wait – standard VAS: 1xx Assets, 3xx Liabilities, 4xx Equity)
- Income Statement: Multi-step (revenue → gross profit → operating → net profit); separate financial/admin expenses
- Cash Flow: Indirect method; reconcile net profit to operating cash
- Notes: Include mandatory disclosures (policies, estimates, contingencies, biological assets changes per 215/2295, GloBE top-up via 82112)
- All reports: Add header/footer with company info, period, generation date/user

## 7. Testing & Quality
- Unit test query handlers (data aggregation logic)
- Integration test: Render report → assert PDF contains expected text (use libraries like PdfPig if needed)
- Manual: Preview in browser, export PDF, check against official VAS templates
